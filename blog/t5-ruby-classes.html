Class in Ruby

7/21/15

A class in Ruby is a collection of methods that you would like to apply to new objects. You can create instance variables within your class that speak across methods and change from object to object. If you want to see the result of the new object you have created, simply ask Ruby to print your new object and call whichever method you would like to see.

In this case, I have created a class named Yarn to hold information about my ever-growing yarn stash. I want to see the color and fiber for each new yarn I have.
To do this I set my instance variables as @color and @ fiber.
When I add a new yarn, and specify what color and fiber it is, Ruby will be able to return those details for each new yarn added. 

class Yarn  
  def initialize(color, fiber)  
    # Instance variables  
    @color = color  
    @fiber = fiber  
  end  
  
  def lookup  
    puts 'You have new yarn!'  
  end  
  
  def display  
    puts "Your new yarn is #{@color} and made of #{@fiber}"  
  end  
end  
  
# making new objects  
yarn_1 = Yarn.new('green', 'wool')  
yarn_2 = Yarn.new('blue', 'alpaca')
yarn_3 = Yarn.new('orange', 'synthetic fiber')

# Display info from lookup method
puts yarn_1.lookup
# Display info from display method 
puts yarn_1.display
puts yarn_2.display
puts yarn_3.display

I have created a new Yarn object outside of the class yarn_1 = Yarn.new('green', 'wool')
yarn_2 = Yarn.new('blue', 'alpaca')
yarn_3 = Yarn.new('orange', 'synthetic fiber')
Underneath the new objects, I have asked Ruby to show the information for the new yarn and the method that has been applied.
Once I run this program it returns 

You have new yarn!

Your new yarn is green and made of wool

Your new yarn is blue and made of alpaca

Your new yarn is orange and made of synthetic fiber

I can do this for however many new yarns I would like to create, as long as I have include the color and fiber instance variables it will always give me that information back.
