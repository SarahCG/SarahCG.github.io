JavaScript and Ruby can have similar behaviors. One example of this is Ruby Hashes vs. JavaScript Objects. The syntax is somewhat similar.

Here is an example of a Hash in Ruby. 

myhash = { key1: "value_1", key2: "value_2", key3: "value_3"}


myhash[:key1] => value_1

Notice that the has contains key value pairs.
Each key (key1, key2, key3) is assigned a value (value_1, value_2, value_3). When you call on the key, it will access it's value.
Pretty cool, right? It's like when you look up a word in a dictionary (the word is the key in this case) to find it's definition (the definition would be the value).

In JavaScript objects behave similar to hashes.
Here is an object in JavaScript

var myObject = { 
thing1:"description_of_thing1", 
thing2:"description_of_thing2", 
thing3:"description_of_thing3" 
}


console.log(myObject.thing2) => description_of_thing2

Notice again, the key value pairing we have here. Thing1 acts as a like a key that holds the value description_of_thing1. If you want the description of thing2, that's easy to find just lookup myObject.thing2.

Hashes and Objects definitely make life easier when you are keeping track of multiple items and want to be able to find them quickly, even though the syntax looks the same there are some small differences in how you call the search for the keys and values in Ruby and JavaScript.
