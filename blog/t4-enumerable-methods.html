Enumerable Methods

7/12/15

Map is an enumerable method in Ruby. It's pretty cool, and I will explain it to you below.

Map allows you to run an operation on each item in a block in Ruby and returns the result. Map can do this without changing the original block, if you run it as a non destructive.
[1, 2, 3,].map {|n| n + n}
returns
[2, 4, 6]
See what happened there? You have a new array where each item added itself to itself. You still have your original array in tact, but now your new array displays the result of what happens when you run the operation
If you want to permanently change the items in an array you can use map!
That little exclamation point there means, "Hey, beware, I am going to make changes to this thing!"
[1, 2, 3,].map! {|n| n + n}
returns
[2, 4, 6]
But now, the array that you are working with is not [1, 2, 3] anymore.
The array keeps the values that you map!ed into it, so until you decide to change it again, your array will be [2, 4, 6].

Don't confuse map with each. I did that while coding and it really threw me for a loop (no pun intended.. HA!). It's good to remember that each just returns each item in your block. So [1,2,3].each
gives you
[1,2,3]
Each doesn't touch anything, it simply tells you what each item is. Map, actually applies an operation to each item and keeps it or not, depending on that destructive !.
